## TOUCAN.EE — Basic Marketing List (GitHub Issues)

1. **Define target student persona**
   - Choose one target student: beginner animation / game-dev learner who feels stuck.
   - Add a short persona description and key goals.
   - **Acceptance criteria:** Persona written in 3–5 sentences.

2. **Clarify core promise**
   - Draft one promise: “I help beginners get unstuck and finish projects.”
   - Adjust wording to match persona language.
   - **Acceptance criteria:** Final promise approved and saved.

3. **Select and commit to platform**
   - Use LinkedIn as the primary platform.
   - Update any internal notes or docs to reflect this.
   - **Acceptance criteria:** Platform selection documented.

4. **Set posting cadence**
   - Plan for 3 posts per week on LinkedIn.
   - Create a lightweight posting calendar or reminder.
   - **Acceptance criteria:** Calendar/reminder exists for weekly posts.

5. **Create 3-post content framework**
   - Structure posts around: Pain → Insight → Light proof.
   - Draft one post for each category.
   - **Acceptance criteria:** 3 drafts complete (one per category).

6. **Daily comment routine**
   - Comment daily for 10 minutes on beginner animation / Unreal posts.
   - Define a simple tracker for consistency.
   - **Acceptance criteria:** Tracker created and first week planned.

7. **Conversation flow**
   - Define response flow: Reply → help briefly → offer 1-on-1 session.
   - Draft example responses for common questions.
   - **Acceptance criteria:** Conversation script ready.

8. **Single offer definition**
   - Offer only: 60-minute mentoring session.
   - Write offer description, price (if applicable), and outcomes.
   - **Acceptance criteria:** Offer copy finalized.

9. **Simplify website content**
   - Keep site simple with sections:
     - Who it’s for
     - What problem you solve
     - How sessions work
     - Contact / message CTA
   - **Acceptance criteria:** Website content updated to match.

10. **Collect 1-line feedback after sessions**
   - Ask for a single-line testimonial after every session.
   - Create a template request message.
   - **Acceptance criteria:** Template ready and in use.

11. **Repost proof after sessions**
   - Share quotes and session insights.
   - Build a reusable repost format.
   - **Acceptance criteria:** 2 example reposts drafted.
