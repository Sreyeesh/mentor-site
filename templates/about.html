Full-stack developer building with AI.

I'm Sreyeesh Garimella. I build web applications, developer tools, and pipelines — and I use AI as a core part of how I work, not an afterthought.

My background is in production: animation, VFX, and game development at studios like Disney, Blizzard, and DNEG. That shaped how I think about software — systems need to be reliable, pipelines need to be maintainable, and tools need to get out of the way of the work.

How I use AI in development

AI has changed how I write and ship code. I use large language models throughout the development process — for drafting, reviewing, and refining — but the judgment about architecture, tradeoffs, and what to actually ship stays with me.

I treat AI as a pair programmer that never gets tired. It speeds up the boring parts — boilerplate, documentation, test scaffolding — so I can spend more time on the parts that require real thinking: system design, edge cases, and making things maintainable long-term.

What I build

Mostly web applications and internal tools: Flask and Python on the backend, straightforward HTML/CSS/JS on the front, deployed to wherever it needs to run. I care about keeping things simple enough that they can actually be maintained — not architectured into oblivion.

I also work on developer tooling: pipeline automation, scripting, build systems. Anything that makes the development loop faster and more predictable.

This site

This is where I write about what I'm learning and building. Mostly notes to my future self, occasionally useful to others. If something here is useful to you, I'm glad.
